# Памятка по запросам

## Активные режимы поиска

- `name`
  - пример: `aspirin`
- `cid`
  - пример: `2244`
- `smiles`
  - пример: `CC(=O)OC1=CC=CC=C1C(=O)O`
- `inchikey`
  - пример: `BSYNRYMUTXBXSQ-UHFFFAOYSA-N`
- `formula`
  - пример: `C9H8O4`

В ручном интерфейсе сейчас доступны только первые три режима. Типизированный backend и агентный режим также умеют `inchikey` и `formula`.

## Активные операции

- `property`
  - лучший первый запрос для карточки соединения
- `record`
  - подходит, когда нужен более полный сырой ответ
- `synonyms`
  - полезно для проверки альтернативных названий

## Что пока только в планах

- `fastformula`
- `fastidentity`
- `fastsimilarity_2d`
- `fastsubstructure`

Это ещё не элементы текущего UI.

## Как агентный режим интерпретирует запрос

- `CID ...` -> `cid`
- чистое число -> чаще всего `cid`
- строка вида InChIKey -> `inchikey`
- строка, похожая на формулу -> `formula`
- строка, похожая на структуру -> `smiles`
- всё остальное -> попытка поиска по `name`
- абстрактные описания свойств -> возврат неясностей и ожидание подтверждения
