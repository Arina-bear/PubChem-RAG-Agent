# QA-отчёт

Дата проверки: 26.03.2026

## Что проверено

- структура репозитория и отсутствие пустых лишних директорий
- backend: тесты, запуск, проверка здоровья сервиса, запросы `/api/query` и `/api/interpret`
- frontend: production build, маршруты App Router, ручной режим, агентный режим
- браузерный end-to-end прогон: вкладки `Обзор`, `Синонимы`, `JSON`
- сетевое поведение браузера: frontend обращается только к своим `/api/*`, а не напрямую к PubChem
- UX и edge-case сценарии: пустой ввод, `no match`, невалидный `SMILES`, абстрактный текстовый запрос, mobile layout
- документация и пользовательские тексты: упрощены и переведены на русский
- `docker compose`: сборка образов, запуск сервисов и smoke-check контейнеров

## Что было исправлено во время QA

- исправлен `backend/Dockerfile`: теперь копируется `src/`, а запуск идёт с `--app-dir src`
- удалены пустые и дублирующие папки, которые мешали корректному распознаванию frontend App Router
- интерфейс и тексты backend-ошибок переведены на русский язык
- документация в `readme-api.md`, `docs/` и `backend/README.md` упрощена и переписана на русском
- добавлена иконка приложения, чтобы убрать `404` на favicon и очистить консоль браузера
- приведены к русскому языку остаточные package docstring'и и метаданные backend
- добавлены `.dockerignore` для backend и frontend, чтобы в Docker не уезжали локальные артефакты
- улучшена интерпретация естественного языка: команды вида `найди aspirin` теперь правильно очищаются до названия соединения
- исправлен UX в агентном списке кандидатов: выбранный пользователем вариант больше не подменяет собой блок `Наиболее вероятная интерпретация`
- упрощена ручная форма: подпись поля ввода теперь зависит от режима (`name`, `cid`, `smiles`), а названия операций показаны по-человечески
