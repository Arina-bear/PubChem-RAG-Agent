# Границы продукта

## Что это за MVP

- Прямой слой запросов к PubChem без векторной базы и без embeddings.
- Интерфейс для поиска соединений.
- Агентный режим, который помогает перевести текст в структурированный запрос.

## Чем этот MVP не является

- Это не классический RAG.
- Это не автономный химический copilot.
- Это не полноценная multi-domain рабочая станция по PubChem.

## Основные пользовательские сценарии

### Ручной режим

- выбрать домен `compound`
- выбрать `input_mode`
- выбрать `operation`
- отправить запрос
- посмотреть нормализованный результат
- при необходимости открыть сырой `JSON`

### Агентный режим

- описать, что нужно найти
- посмотреть 1–3 кандидата запроса
- явно выбрать вариант
- запустить запрос
- посмотреть тот же результат, что и в ручном режиме

## Текущие ограничения

- только `compound`
- frontend не обращается к PubChem напрямую
- публичный API ограничен `health`, `query`, `interpret`
- в ручном UI доступны только `name`, `cid`, `smiles`
- backend и агентный режим дополнительно умеют `inchikey` и `formula`
- в интерфейсе активны вкладки `Обзор`, `Синонимы`, `JSON`
- `bundle`, `autocomplete`, `jobs`, `PUG View` и тяжёлые поиски пока в следующем этапе
